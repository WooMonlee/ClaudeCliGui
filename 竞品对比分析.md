## 逐项说明

### 易用性（10/10）
单文件 70MB，双击即用。一键装环境。右键菜单集成。清零卸载。**只有我们能给完全不懂命令行的用户用。**

### 忠于原版（7/10）
思考面板已折叠，流式输出实时。缺：终端色、原地刷新动画。用 PTY 的两个竞品（CloudCLI、LKbaba）在这里满分。

### 复制/表格（10/10）
**WPF FlowDocument 唯一能复制表格到 Word 保留格式的方案。** 其他竞品全部是纯文本/HTML 渲染，复制表格到 Word 就是 ASCII 乱码。

### 零门槛部署（10/10）
**唯一自动安装 Node.js + Claude CLI 的方案。** 其他竞品全部假设用户已装好环境。

### 文件管理（5/10）
无文件树、无文件浏览器。CloudCLI 有完整的文件树 + 语法高亮编辑器。

### Git 集成（0/10）
零 Git 功能。CloudCLI 有 Stage/Commit/分支切换/Diff。

### MCP 管理（2/10）
只支持手动编辑 settings.json。LKbaba 有完整的图形化 MCP 配置界面。

### API 配置（8/10）
有提供商预设（DeepSeek/Anthropic/自定义）、模板自动填充。缺：多 Profile 切换（nbplugins 支持）。

### 多代理支持（1/10）
仅 Claude Code CLI。CloudCLI 支持 Claude Code + Cursor + Codex + Gemini。

### 跨平台（2/10）
仅 Windows。CloudCLI 全平台（Web 应用 + PWA），这是 Web 方案的天然优势。

### 成熟度（4/10）
v1.0 刚发布，社区零用户。CloudCLI 1.1 万 Star，LKbaba 功能最丰富。

## 定位

三个竞品都是"给**已经装了 Claude Code CLI 的开发者**的增强工具"。
我们是"给**任何 Windows 用户**的 Claude Code CLI **入口**"。

他们的用户：会装 Node.js、会用终端、配过环境变量。
我们的用户：双击 exe → 点"一键安装"→ 开始用。

差异化足够大，赛道不同。成熟度靠时间和用户积累。
