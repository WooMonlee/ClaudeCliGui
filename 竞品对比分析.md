# Claude Code GUI 竞品对比分析

> 调研时间：2026-07-11
> 对比对象：ClaudeCliGui (我们) vs 4 个开源竞品

---

## 竞品一览

| 项目 | Stars | 技术栈 | 形态 | 定位 |
|------|-------|--------|------|------|
| [siteboon/claudecodeui](https://github.com/siteboon/claudecodeui) | ~11,800 | React + Express + SQLite | Web 应用 | 全功能 Web UI |
| [LKbaba/Claude-code-ChatInWindows](https://github.com/LKbaba/Claude-code-ChatInWindows) | ~63 | TypeScript | VS Code 插件 | IDE 内 GUI |

| [nbplugins/NetbeansPluginClaudeCodeGui](https://github.com/nbplugins/NetbeansPluginClaudeCodeGui) | — | Java | NetBeans 插件 | IDE 内 GUI |

---

## 一、siteboon/claudecodeui（CloudCLI）— 最成熟

### 他们的优势

| 优势 | 详情 |
|------|------|
| **全平台 Web 界面** | React + WebSocket，手机/平板/桌面都能用，PWA 可安装到桌面 |
| **内置文件浏览器** | 文件树 + 语法高亮编辑 + CRUD，不离开界面就能改代码 |
| **Git 集成** | Stage/commit/分支切换/差异查看，UI 内完成 |
| **会话管理** | 自动发现 `~/.claude` 会话、跨设备同步、重命名、导出 |
| **MCP 服务器管理** | 图形化增删改 MCP 服务器，同步到 Claude Code 配置 |
| **插件系统** | 10+ 插件：Token 计算、任务看板、会话监控、定时任务 |
| **多 Agent 支持** | Claude Code + Cursor CLI + Codex + Gemini CLI |
| **REST API** | 完整 API，可编程调用 |
| **国际化** | i18n 多语言支持 |

### 他们的劣势

| 劣势 | 详情 |
|------|------|
| **需要 Node.js 环境** | 必须先装 Node.js v22+，不是单文件 |
| **Web 服务依赖** | 需要启动后端服务器，占用端口 |
| **无环境部署功能** | 不会帮你装 Node.js 或 Claude CLI |
| **无右键菜单集成** | 没有 Windows 资源管理器右键快捷入口 |
| **体积大** | 前端 + 后端 + 数据库，npm 包较重 |

## 二、LKbaba/Claude-code-ChatInWindows — 功能最全

### 他们的优势

| 优势 | 详情 |
|------|------|
| **Windows 原生免 WSL** | 直接 Git Bash + Node.js，不需要 WSL |
| **MCP/Skills/Hooks 全套 GUI** | 图形化管理 MCP 插件、26 种 hooks 事件、Skills 系统 |
| **第三方 API 配置界面** | 支持 anyrouter、tuziapi、Kimi K2 等国内 API |
| **实时 Token/费用追踪** | 界面上实时显示 token 消耗和费用 |
| **多模态** | 支持粘贴/插入图片作为视觉输入 |
| **交互式选项卡片** | `ask` 块渲染为可点击按钮 |
| **内嵌 AI 助手** | Gemini + Grok 助手面板 |
| **PTY 交互式驱动** | 使用真实交互式 CLI 会话 |

### 他们的劣势

| 劣势 | 详情 |
|------|------|
| **必须装 VS Code** | 是 VS Code 插件，不能独立运行 |
| **不帮装 Node.js** | 用户需自备 Node.js 和 Claude CLI |
| **无项目管理** | 没有独立的项目列表/切换（依赖 VS Code 工作区） |
| **无右键菜单** | 不能右键文件夹直接开始对话 |
| **学习门槛高** | 功能多但配置复杂，不适合快速上手 |

## 三、nbplugins/NetbeansPluginClaudeCodeGui — IDE 集成

### 他们的优势

| 优势 | 详情 |
|------|------|
| **IDE 内无缝工作** | 直接在 NetBeans 里对话，无需切换窗口 |
| **图形化 Diff** | 每次文件变更前预览、接受/拒绝 |
| **多 Profile** | 不同项目用不同 API Key 和配置 |
| **OpenAI 兼容代理** | 支持 DeepSeek、Groq、Ollama 等任意兼容端点 |
| **Prompt 收藏** | Ctrl+Up/Down 历史搜索，保存 favorites |
| **文件附件** | `@path` 自动补全、拖拽、剪贴板粘贴 |

### 他们的劣势

| 劣势 | 详情 |
|------|------|
| **必须装 NetBeans** | 绑定特定 IDE，不是独立应用 |
| **不帮装环境** | Node.js / Claude CLI 需用户自行安装 |
| **无右键菜单** | 没有系统级集成 |
| **用户群小** | NetBeans 用户远少于 VS Code |

## 四、ClaudeCliGui（我们）— 竞品对比总结

### 我们的独特优势

| 优势 | 说明 | 竞品谁有 |
|------|------|---------|
| **单文件 exe，免安装** | 70MB，复制即用 | ❌ 全部没有 |
| **一键部署环境** | 启动即检测，自动下载 Node.js + 安装 Claude CLI，国内镜像 | ❌ 全部没有 |
| **免 CC-switch/代理** | 内置国内镜像源，无需任何网络工具 | ❌ 全部没有 |
| **右键菜单集成** | 右键任意文件夹直接开始对话 | ❌ 全部没有 |
| **清零卸载** | 一键清除所有痕迹 | ❌ 全部没有 |
| **自动更新** | 后台 GitHub Release 检测，静默下载 | ⚠ CloudCLI 有 |
| **快照/历史上滚** | 对话持久化，上滚加载更多 | ⚠ CloudCLI 有 |

### 我们的劣势

| 劣势 | 说明 | 谁做得更好 |
|------|------|-----------|
| **无文件浏览器** | 不能在不离开界面的情况下浏览/编辑文件 | CloudCLI |
| **无 Git 集成** | 不能 Stage/commit/分支切换 | CloudCLI |
| **无 MCP 图形管理** | MCP 配置需手动编辑 JSON | LKbaba、CloudCLI |
| **无 IDE 集成** | 不能嵌入 VS Code/NetBeans | LKbaba、nbplugins |
| **无多 Agent 支持** | 仅支持 Claude Code CLI | CloudCLI |
| **无 Token 实时统计** | 界面不显示 token 消耗 | LKbaba |
| **仅 Windows** | 不支持 macOS/Linux | CloudCLI |
| **仅 DeepSeek API** | 只支持 DeepSeek 端点 | nbplugins（OpenAI 兼容）、LKbaba |
| **非 Web 界面** | 不能用手机访问 | CloudCLI |

## 定位建议

ClaudeCliGui 的差异化优势在于 **"零门槛上手"**——四个竞品都假设用户已经有 Node.js 和 Claude CLI，我们是从零开始全自动。这个定位在中文 Windows 用户群（不熟悉命令行、网络受限）里是独一无二的。

后续可以补的方向（按优先级）：
1. **文件浏览器** — 侧边栏加简单文件树
2. **Token 统计** — 解析 result JSON 的 usage 字段显示到状态栏
3. **多 API 端点支持** — 不只 DeepSeek，加 Anthropic/Ollama 等
4. **WebSocket 远程访问** — 复刻 Web 版的移动端访问能力
